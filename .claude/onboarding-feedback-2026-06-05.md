Source: two voice recordings of a live onboarding + authed-app walkthrough with a prospective
clinician (Dr. Martin), Q driving the demo. Auto-transcribed (noisy; background chatter filtered out).
Two passes: (1) onboarding flow, (2) deeper authed app (Home → Overview → My Profile → Settings →
Campaigns → Integrations/Buffer → Members).

## What landed well (keep / don't regress)
- Organic, human-feeling interview ("you were thinking, emoting, like a real conversation").
- Brand kit — "nailed it… you even got this part."
- Clean / simple aesthetic — "I always prefer clean… this captures that."
- **AI cost transparency** ($9 for a month of interviews) — strong positive vs AI's bad press;
  "this is not resource intensive… good thing to see."
- Pre-filtered Gmail search during Buffer setup — "holy crap, that's awesome."
- Members & roles invite-by-email + access matrix — worked smoothly.
- Tone controls (soften edges, smart defaults) — liked.

## Action items

### P0 — legibility & contrast (recurring "barely invisible")
- "real answer, real output — nothing made up for the demo" tagline faded into background; make prominent.
- Multiple elements "I can't see this / barely invisible" — audit low-contrast text.
- Default font too small app-wide ("everything feels shrunken"); also flagged on sign-up screen.

### P0 — affordance confusion (looks clickable / doesn't look like a control)
- Channel picker: checkboxes don't highlight on select → clickable boxes that change color. Too much
  white space → 2–3 columns with icons/graphics, responsive on mobile.
- Home screen feature tiles read as clickable buttons but are capability descriptions — distinguish
  "features it can do" from "things you click."
- Sign up / Sign in toggle doesn't look like a toggle (looks like buttons) and is redundant where
  placed ("sign up does nothing, you're already on sign up"); make it read as a toggle, by/above Sign In.

### P1 — plain-language copy (jargon doesn't land with clinicians)
- Buffer "fans social post" — "fans" meaningless to non-tech users; find synonym.
- Campaign description field too technical → plain explanation (campaigns guide content toward a
  goal/event, e.g. a seminar; your voice stays your voice).
- Buffer "Read full description" hyperlink → "Get your Buffer account / Click here"; explain Buffer does
  the integrations, you just connect to it (no duplicate work).
- Location field ambiguous → clarify it's a nickname/short name for the location.

### P1 — naming & redundancy (profile/settings)
- Overview greeting uses username, not the display name given ("Dr. Martin") — use the
  "how do patients refer to you?" answer.
- Voice section appears twice on My Profile — consolidate.
- Display-name editing in Settings mildly confusing ("my first name is Doctor?").

### P2 — feature ideas
- Brand voice from patient reviews (not just self-description); must ignore fake/1-star reviews.
  "I'd never talk about myself the way my patients do."
- Channel-selection guidance at onboarding: explain how to pick, recommend limiting to channels you'll
  actually post to (don't enable YouTube/TikTok/Shorts with no video — "adds visual load"). Smart default
  here: newsletter + GBP post + Instagram feed/reels.
- Buffer API key step: "you won't see this again — save in 1Password" prompt.
- Overnight batch updates: run version/content updates overnight in one pass.
- Reinforce video script/talking-points capability (15-sec hook, reel-length guidance) — visible interest.
