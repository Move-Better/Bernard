What changed vs today — each maps to your click-through notes

1 · Output type is explicit

Today: a static “Instagram Carousel” badge — you couldn’t tell Post vs Story vs Reel, or switch.

Now: a clickable pill — Instagram Post / Carousel / Story / Reel (and the same pattern for every platform with multiple output types).

2 · Photo is full-bleed by default

Today: the “Dark Claim” theme shrank the photo into the top, with a navy text-card band below — it looked “scaled to fit.”

Now: the photo owns the whole slide; text sits on top with a soft scrim. The navy claim-card is one optional layout, not the default.

3 · Preview is a phone mockup that matches the render

Today: a full-screen overlay that drew a different CSS approximation — not what publishes.

Now: an IG phone frame showing the real baked slide — “how people actually see it.” Phone/Desktop toggle for LinkedIn & Facebook.

4 · One photo control

Today: “Swap / add a photo” and a separate “Use an attached photo” dropdown — two ways to set one slide’s photo.

Now: one panel — the slide’s photo, with Replace (AI picks / library / upload / describe). The redundant dropdown is gone.

5 · Per-slide model

Today: media is attached to a pool, then bound per slide — redundant.

Now: pick a photo → it lands on the slide (attach + bind in one step). + Add = a blank slide that prompts “Add a photo.” Reuse across slides OK; deleting a slide drops its photo.

Parked (your call)

Bulk multi-select (long carousels) — noted for later; we make “a few slides” great first. Real font picker stays in the Brand Kit (text revision is U2.2, next).
