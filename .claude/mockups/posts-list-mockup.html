Where a one-off “Post” lives — the Posts tab

The shipped Post composer creates posts with no interview, but nothing in the app lists them, so a saved draft is currently unreachable. This adds a Posts tab on Stories — a real home with the Draft → Scheduled → Published lifecycle. Each row opens the existing editor + publish flow.

New tab on Stories Drafts you can finally reach Scheduled visible in-app Published history

Fixes

The orphaned-draft bug. Today a Post saved as a draft can’t be found anywhere — buildStories() drops it because it has no interview. This tab lists it by workspace + status, so it’s reachable, editable, and publishable.

Each row is one post you wrote once — the chips show the channels it went to. One message, one row.

Opens

Click any row → the existing editor + publish flow (/publish/:id, already works by id). A text post opens the text editor, a photo/video opens the media editor — so “route media to the editor” from earlier is handled here, in one place, for every post.
