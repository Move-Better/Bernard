What changed here — P3
  • Decay is now live (“Slipping in rank”). Rule, calibrated to real Move Better data: was in the top 20, dropped ≥3 positions week-over-week, ≥10 impressions. The generic impr ≥ 50 / pos ≤ 10 default flagged zero queries here — a small clinic's non-branded terms all sit under 50 impressions — so the floor is volume-appropriate. The 3 cards above are Move Better's real slippers this week.
  • “Did it work?” is new — post-publish ranking delta. Website/blog pieces only (social doesn't move Google rank). Needs a snapshot from before the publish date, so it fills going forward, not retroactively. Attribution uses a new content_items.target_query stamped from the ?topic= seam at creation (exact match), with the existing queryMatchesTopic word-overlap as a fuzzy fallback for older pieces — labeled “Exact” vs “Likely” on each card.
  • Cannibalization stays locked — it needs per-URL data we don't store yet (snapshots are query-level). This PR adds a page column + fetches dimensions:['query','page'] so it starts accruing Mon Jul 13; the card lights up ~Jul 27. Can't be backfilled.
  • Two fixes rolled in: the week counter now buckets by ISO week (not calendar date, which a mid-week re-run inflated), and the locked cards now read the API's locked.*.ready flag instead of being hardcoded, so they unlock on their own.
  • Loop closes into cadence: a confirmed “it worked” / “it slipped” feeds project-adaptive-cadence as a signal — publish more like what's working.
