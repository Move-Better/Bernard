# Bernard UI Audit — 2026-06-22

**Method:** Live page-by-page sweep of prod (`movebetter.withbernard.ai`, SHA `0bc59567`) via Q's logged-in Chrome, plus a parallel source-side `ui-reviewer` pass. Cataloging only — no code changes this pass. Three axes: **language/copy**, **flow/IA**, **color/visual**.

**Severity:** P0 = broken or confusing enough to lose/mislead the user · P1 = clear papercut · P2 = polish.

**Finding IDs:** `<axis>-<page><n>` — e.g. `L-H1` = Language, Home, #1.

---

## Summary — themes & priorities

The app is **well-designed and differentiated** screen-by-screen (the autonomy ladder, team streaks, moment-first feed, the living "Book", cross-staff knowledge map are all genuinely good). The issues are almost all **consistency seams between screens** — the kind you only catch flipping page to page, which is exactly this exercise.

**Theme 1 — One product, many vocabularies (the dominant issue; SYS-1, SYS-2, L-G3).** The same thing is named differently on adjacent screens:
- Captured object: **interviews** (Home, profile) vs **stories** (Stories, Overview).
- Pipeline/status: Storyboard / Publish / Distribute / "out the door"; and statuses `Draft` / `Drafting` / `Capture` / `Scheduled` / `needs draft` / `Content ready` / `went live`.
- People: **clinician / staff / author**.
- Surface names: **Analytics / Performance / Insights**; **Create → New**; **Channels → Presence**; **My profile → Staff profile**; a stray "Dashboard".
- Help link: "How it works" vs "Help / How this works".
> **Highest-leverage fix:** agree a vocabulary canon, then one editor pass. Unblocks ~15 findings.

**Theme 2 — Amber is losing its meaning (SYS-3, C-O1, C-A1, C-H1, C-W1).** `--action` amber is reserved for "do this now", but it's also used decoratively (page-header gradients, flat "% engaged" badges, the "assembling" status), and there's a near-duplicate `--warning` token plus `--info` (sky blue) leaking into action surfaces. Net: the eye no longer trusts amber = act now.

**Theme 3 — Document-title bugs (SYS-5).** "Ads · Bernard · Bernard" (doubled) and "Move Better — Bernard" (Stories, Book — missing page name). Root cause found: pages that don't set a title fall back to the workspace App name.

**Theme 4 — A handful of real copy bugs.** Truncated "Cross-staff concept coverage **for** ___" (Knowledge synthesis, P1), "since last **regen**" (Book), "**Los Angeles times**" shown to a Portland clinic (Your week), "**Validate** your words" (ambiguous CTA), "**Aimed** by AI".

**Theme 5 — Three overlapping dashboards (F-G1).** Home / Overview / Analytics repeat publish-count and run-cost with inconsistent time windows that don't reconcile.

**Theme 6 — Small flow papercuts.** Double status pills (Your week), unlabeled metadata dropdowns (Story detail), no clear primary action (Moment Miner cards), delete beside primary (profile), filter overload (Stories).
